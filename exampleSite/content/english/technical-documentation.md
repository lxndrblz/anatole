Bad documentation costs time. Good documentation saves it.

I write technical docs and build help centers with two principles: clarity first, built to scale.

I've written user manuals, quick start guides, SOPs, conceptual guides, API/SDK docs, technical blogs, best practices, and diagrams. Beyond writing, I've also scripted, created, edited, and voiced training tutorials.
